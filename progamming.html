The Role of Computing in Biomedical Engineering

In modern healthcare, modern medicine intersects closely with software and computing hardware. This document outlines the structured integration of Programming in C and Python throughout the 4-Year Bachelor of Engineering program at Omani University. Students develop foundational programming logic, execute real-time low-level device control via C firmware, and apply data analysis, biomedical signal manipulation, and artificial intelligence models using Python ecosystem standards.

Detailed Programming Syllabus Across Semesters

Semester I: Introduction to Computing & Algorithmic Logic

Python Core
Course Code: COMP 1101 Credits: 3 (L: 2, T: 0, P: 2) Pre-requisites: None
Theoretical Units Introduction to computational thinking; Variables, Expressions, and Built-in Data Types; Control Structures (Conditional branches and loops); Functions, Scoping, and Modular structural programming.
Practical Lab Modules Writing Python scripts to parse simulated textual patient logs; Automating unit metrics parsing (e.g., Celsius to Fahrenheit conversions, BMI calculations); Implementation of basic recursion.
Biomedical Context Element: Writing algorithms to screen datasets for basic anomalies, mimicking high-level data pre-sorting logic found in electronic health record (EHR) systems.

Semester II: Engineering Computer Programming for Problem Solving

C Core
Course Code: COMP 1102 Credits: 4 (L: 2, T: 1, P: 2) Pre-requisites: COMP 1101
Theoretical Units Procedural style development in C; Strict typing, Constants, operators, and formatting primitives; Complex control flows (`switch-case`, nested `while`/`for`); Single and Multi-dimensional Arrays; Memory Address space basics, Pointers, and Function arguments execution styles.
Practical Lab Modules Writing string sorting algorithms; Multi-dimensional matrix mathematical execution programs; Dynamic memory allocations tracking via `malloc()`, `calloc()`, and memory cleaning with `free()`.
Biomedical Context Element: Constructing multidimensional arrays intended to store grid values representing localized biosensor reading streams mapped over geometric tissue coordinates.

Semester III: Data Structures & Computational Logic

C & Python Dual Application
Course Code: BMED 2101 Credits: 3 (L: 2, T: 0, P: 2) Pre-requisites: COMP 1102
Theoretical Units Linear structures (Singly, Doubly, and Circular Linked Lists; Stacks and Queues operations architectures); Non-linear structural paradigms (Binary Search Trees, Graph representations); Searching techniques and Sorting sorting algorithms optimizations.
Practical Lab Modules Developing fully functional self-referential linked list node paths in C; Tracking dynamic buffers utilizing Queues; Performance optimization audits tracking time complexity limits.
Biomedical Context Element: Developing structural Queues modeling the FIFO storage pipelines required when handling live biosignal streaming inputs before filtering.

Semester IV: Object-Oriented Software Design

Advanced Python
Course Code: COMP 2201 Credits: 3 (L: 2, T: 0, P: 2) Pre-requisites: COMP 1101
Theoretical Units Object-Oriented Programming (OOP) paradigms; Classes, instance initializations, and structural Methods; Inheritance patterns, Encapsulation rules, and Polymorphism; Advanced exception handling; Modular systems build mechanics.
Practical Lab Modules Building structured class hierarchies with Python; Exception assertion routines parsing continuous text files; Constructing desktop application interfaces (using Tkinter/PyQt framework).
Biomedical Context Element: Modeling clinical hardware component behaviors via virtual software abstractions (e.g., generating an abstract base class `MedicalSensor` from which classes like `Oximeter` and `ECG` inherit).

Semester V: Microcontrollers & Biomedical Instrumentation Interfacing

Embedded C
Course Code: BMED 3101 Credits: 4 (L: 3, T: 0, P: 2) Pre-requisites: COMP 1102 & EE Circuits
Theoretical Units Microcontroller hardware architecture; Low-level Register manipulations using Embedded C; General Purpose Input/Output (GPIO) operational constraints; Analog-to-Digital Conversion (ADC) techniques; Serial Communication interfaces (UART, I2C, SPI); Interrupt service handling routines.
Practical Lab Modules Configuring internal MCU timing behaviors; Programming ADC registers to collect voltage readings from an external analog source; Interfacing hardware peripheral screens using SPI protocols.
Biomedical Context Element: Writing real-time Embedded C code to capture analog signals from a pulse oximeter circuit frontend, digitize the waveform data, and send it over a serial interface.

Semester VI: Digital Biomedical Signal Processing

Python Libraries
Course Code: BMED 3202 Credits: 4 (L: 3, T: 1, P: 2) Pre-requisites: BMED 2101 & Signals Systems
Theoretical Units Discrete-time signal operations; Sampling theorem compliance; Discrete Fourier Transform (DFT) and Fast Fourier Transform (FFT) theory; Digital filter architectures (FIR and IIR filter structures); Noise attenuation and feature tracking algorithms.
Practical Lab Modules Using Python data libraries (`NumPy`, `SciPy.signal`, `Matplotlib`) to implement digital filtering systems; Removing baseline wander and powerline noise from real, unfiltered patient ECG records.
Biomedical Context Element: Building automated data-cleaning pipelines in Python to remove environmental electrical noise from electromyogram (EMG) or electrocardiogram (ECG) sensor traces.

Semester VII: Medical Image Processing & Healthcare Artificial Intelligence

Python Ecosystem
Course Code: BMED 4101 Credits: 4 (L: 3, T: 0, P: 2) Pre-requisites: BMED 3202
Theoretical Units Digital medical imaging structure paradigms (DICOM file specs); Spatial filters, morphological segmentation, edge detection mechanisms; Classical Machine Learning frameworks; Artificial Neural Networks and Convolutional Neural Networks (CNN) for imagery structures processing.
Practical Lab Modules Processing raw imaging slices with OpenCV and Scikit-Image; Extracting tumor geometries via automated boundary masks; Building simple classifier models using Scikit-Learn.
Biomedical Context Element: Designing a custom CNN pipeline in Python to identify structural abnormalities in lung X-rays or classify tissue anomalies in brain MRI scans.

Semester VIII: Capstone Graduation Project & System Deployments

Full Deployment (C + Python)
Course Code: BMED 4202 Credits: 6 (L: 0, T: 0, P: 12) Pre-requisites: All Prior Technical Blocks
Theoretical Units Full-stack embedded-to-cloud software engineering; Code profiling, optimization, and documentation practices; Medical software lifecycle compliance frameworks (IEC 62304 standards); Inter-language bindings (running C optimization subroutines inside Python wrapper blocks).
Practical Lab Modules Developing and debugging a cross-platform prototype; Implementing structural firmware pipelines alongside diagnostic Python analytics engines; Writing rigorous integration tests.
Biomedical Context Element: Designing a functional medical device prototype (e.g., a wearable biosensor system using Embedded C for data gathering and a Python GUI dashboard for cloud analytics and medical diagnostics).

Comprehensive 4-Year Academic Study Plan

Phase 1: Computational Thinking & Procedural Foundations (Years 1)

Goal: Transition from programmatic ignorance to absolute code tracking literacy across procedural environments.

  • Weeks 1 - 8 (S1): Master Python primitives, flow paths, loops, and list manipulation mechanics. Omit complex algorithms and prioritize writing functional code blocks.
  • Weeks 9 - 16 (S1): Focus on data collection paradigms, dictionary lookups, and basic text file parsing containing mock vitals data.
  • Weeks 17 - 24 (S2): Re-evaluate execution logic through strict typing conventions in C. Practice writing correct memory layout syntax manually.
  • Weeks 25 - 32 (S2): Master pointer structures, memory addresses, allocations, and manually configure complex structural arrays.
Phase 2: Structural Cleanliness & Enterprise Objects Foundations (Year 2)

Goal: Learn to choose efficient data layouts and object relationships for complex software applications.

  • Weeks 33 - 40 (S3): Implement dynamic structures (Singly and Doubly Linked Lists) in C. Trace memory allocations step-by-step to prevent leaks.
  • Weeks 41 - 48 (S3): Build recursive trees and custom sorting routines. Evaluate algorithm execution costs using Big-O time complexity models.
  • Weeks 49 - 56 (S4): Transition back to Python using Object-Oriented paradigms. Model sensor payloads using classes, attributes, methods, and inheritances.
  • Weeks 57 - 64 (S4): Learn to build standard desktop application interfaces using Tkinter or PyQt for real-time visualization of mock data feeds.
Phase 3: Hardware Interfacing & Analytical Signal Implementations (Year 3)

Goal: Bridge software with actual physical medical hardware instrumentation components.

  • Weeks 65 - 72 (S5): Study target MCU register maps. Write low-level Embedded C scripts to control hardware pins and respond to external interrupts.
  • Weeks 73 - 80 (S5): Interface SPI/I2C communication chips and program the ADC system to fetch analog sensor waves cleanly.
  • Weeks 81 - 88 (S6): Load digital signal wave series array objects into Python using NumPy. Master array transformations and vector operations.
  • Weeks 89 - 96 (S6): Apply SciPy filter modules to extract clean clinical bio-wave signals from heavily corrupted raw data recordings.
Phase 4: Advanced Intelligent Classifiers & Medical Deployments (Year 4)

Goal: Deploy production-ready, AI-driven medical device prototypes that follow international design standards.

  • Weeks 97 - 104 (S7): Process medical imagery structures (DICOM files) using OpenCV. Apply morphology transforms to segment tissue boundaries and isolate target tumors.
  • Weeks 105 - 112 (S7): Train Scikit-Learn classifiers and build Convolutional Neural Networks (CNNs) to automate disease screening tasks.
  • Weeks 113 - 128 (S8): Develop your final capstone project. Build real-time data-acquisition systems using Embedded C and combine them with Python analytics and visualization suites. Maintain clean git commit histories and verify system safety profiles.
